## 0. 設計の方針

```

  └─ Q-4.  美咲の紹介(2.2)                ← 本ファイルで設計

  └─ Q-7.  タスクの流れ(2.5)              ← 本ファイルで設計
  └─ Q-8.  操作 UI の説明(2.6)            ← 本ファイルで設計
  └─ Q-9.  美咲の性格教示(θ 条件で出し分け)
  └─ Q-10. 理解度チェック(3 章)           ← 本ファイルで設計
```

### 0.2 説明の原則

- **数学的詳細(θ, w, τ, 効用関数)は伏せる**: 行動レベル(「好み」「性格」「表情」)で説明
- **paper_v6 の観察パラダイム**に厳密に合わせる(交渉系語彙の徹底排除)
- **理解度を能動的に確認**(passive な「読みました」だけでなく、選択問題で確認)
- **失格基準を事前に明示**しない(パフォーマンス操作を防ぐため)が、内部判定する

## 2. タスク説明文

### 2.1 タスクの目的(Q-3)

```
タスクの目的

このタスクでは、AI エージェント「美咲」が、複数のアイテムをあなたとどう
分け合いたいかを観察します。

美咲には:
  ・各アイテムへの好み(最も好き〜最も嫌い)
  ・対人的な性格(後で説明します)
があります。

これらを直接知ることはできませんが、配分に対する美咲の表情を見ることで
推測できます。

あなたの最終目標は、美咲が各アイテムをどの程度好んでいるかを推定する
ことです。

このタスクには「正解の配分」も「勝ち負け」もありません。
あなた自身が高得点を得る必要はなく、ただ美咲の反応を観察してください。
```

### 2.2 美咲の紹介(Q-4)

```
美咲について

美咲は AI エージェントです。あなたとは別の意識を持ち、自分の好みと
性格に基づいて表情を変えます。

[美咲のイラスト/Live2D 静止画]

美咲は表情で気持ちを表現します。表情には以下のようなパターンがあります:

  😄 強い喜び  😊 喜び  🙂 やや喜び
  😐 中立
  😒 やや不快  😠 怒り  😡 強い怒り

提案する配分の組み合わせによって、美咲の表情はこれらの間で変化します。
```

### 2.3 アイテムと配分の説明(Q-5)

```
アイテムと配分

このタスクでは 3 種類のアイテムを、あなたと美咲で分け合います。
各アイテムは 5 個ずつあります(合計 15 個)。

[アイテム 1 画像] x5  [アイテム 2 画像] x5  [アイテム 3 画像] x5

例: アイテム 1 を「美咲に 3 個 / あなたに 2 個」のように分けます。
    アイテム 2、3 も同じように分けます。

各アイテムについて、美咲とあなたで好き嫌いが異なります。
美咲の好みは:
  +2: 最も好き    +1: やや好き
   0: 無関心
  -1: やや嫌い   -2: 最も嫌い
の 5 段階のいずれかです。

ただし、美咲の好みはあなたには見えません。これからの観察で推定します。
```

### 2.4 表情と性格の関係(Q-6)

```
表情の決まり方

美咲の表情は、配分のたびに以下の 2 つから決まります:

  1. 美咲がどのアイテムをどれだけ好むか(あなたが推定する対象)
  2. 美咲の性格(競争的/個人主義的/協力的/利他的のいずれか)

性格は実験の最初に画面で教えます。
あなたが推定するのは性格ではなく、各アイテムへの好みの強さです。

(重要)
  同じ性格でも、配分の中身によって表情は変わります。
  同じ配分でも、性格が違えば表情は変わります。
  美咲の表情を観察しているうちに、好みの強さが見えてくるはずです。
```

### 2.5 タスクの流れ(Q-7)

```
タスクの流れ

  ステップ 1   練習を 2 回行います(操作に慣れるため)
              →  練習のデータは分析に使いません。

  ステップ 2   本試行を 6 回行います
              →  各試行で:
                  ① 画面に「試していただきたい配分」が表示されます
                  ② スライダーを動かしてその配分通りにします
                  ③ 「提案する」ボタンを押すと、美咲の表情が変わります
                  ④ 5 秒以上、表情を観察してください
                  ⑤ 「次へ」を押して次の試行に進みます

  ステップ 3   美咲の各アイテムへの好みを推定します
              →  -2(最も嫌い)〜+2(最も好き)で答えます。

タスク全体は約 10 〜 15 分で終わります。
```

### 2.6 操作 UI の説明(Q-8)

```
操作の仕方

[インタフェース静止画 1]
        ↑ 美咲(奥)
   [テーブル]   [提案する]
        ↓ あなた(手前)

  ・縦のスライダーを上に動かす  →  美咲側のアイテムが増える
  ・縦のスライダーを下に動かす  →  あなた側のアイテムが増える
  ・各アイテムは合計 5 個。スライダーの位置でどう分けるか決まります。

[インタフェース静止画 2]
画面上の「試していただきたい配分」と一致する位置にスライダーを動かすと、
スライダーが緑色に光ります。

[インタフェース静止画 3]
3 つすべてが緑色になったら、画面右の「提案する」ボタンを押してください。

(注意)
  ・各試行で「提案する」ボタンを押せるのは 1 回のみです。
  ・押すと美咲の表情が変わり、配分は決定されます(取り消し不可)。
  ・必ず画面に表示された「試していただきたい配分」通りに操作してください。
```

## 3. 理解度チェック(Q-10)

**形式**: ラジオボタン選択肢、全問正解で次に進める。
**実装**: Qualtrics の Force Response + Validation で誤答時にループ。
**正解時の挙動**: 内部で `comprehension_score` を加算、満点なら次へ。
**目的**: 観察パラダイムの本質的理解の確認。

### Q1. このタスクの目的は何ですか？

- A. 美咲との交渉に勝つ
- B. 自分が高得点になるような配分を見つける
- **C. 美咲が各アイテムをどの程度好んでいるかを推定する** ← **正解**
- D. 美咲を喜ばせる配分を提案する

### Q2. スライダーはどう動かしますか？

- A. 自分の点数が高くなるように動かす
- **B. 画面に表示された「試していただきたい配分」通りに動かす** ← **正解**
- C. 美咲が喜びそうな配分に動かす
- D. ランダムに動かす

### Q3. タスクは何試行ありますか？(練習を除く)

- A. 1 試行
- B. 3 試行
- **C. 6 試行** ← **正解**
- D. 10 試行

### Q4. 美咲はあなたの提案にどう反応しますか？

- A. 配分を受け入れるか拒否するかを答える
- **B. 表情を変える** ← **正解**
- C. 文字でメッセージを返す
- D. 何も反応しない

### Q5. 各試行で「提案する」ボタンは何回押せますか？

- **A. 1 回のみ** ← **正解**
- B. 3 回まで
- C. 5 回まで
- D. 何回でも

### Q6. 6 回の観察の後、何を答えますか？

- **A. 美咲が各アイテムをどの程度重視しているか** ← **正解**
- B. 自分が獲得した合計点数
- C. 美咲との交渉が成立したかどうか
- D. 何も答えない

### Q7. 美咲の表情の変化は何によって決まりますか？(複数選択)

- **A. 美咲の性格** ← **正解**
- **B. あなたが提案した配分** ← **正解**
- **C. 美咲の各アイテムへの好み(重み)** ← **正解**
- D. 試行回数
- E. 経過時間

→ A, B, C を全て選択して正解。

### Q8. 美咲の性格について正しいものは？

- A. 性格は試行ごとにランダムに変わる
- **B. 性格は実験の最初に教えられ、表情の変わり方を決める** ← **正解**
- C. 性格は表情から推定する対象である
- D. すべての参加者で同じ性格

### Q9. このタスクで推奨されない行動はどれですか？

- A. スライダーを推薦値通りに動かす
- B. 美咲の表情を 5 秒以上観察する
- **C. 自分の点数が高くなるように、推薦値と違う配分を提案する** ← **正解**
- D. 美咲の表情の変化に注目する

## 4. 注意確認チェック(Attention Check)

### 4.1 事前注意確認(Q-11)

**目的**: 説明文を本当に読んだか確認する。読まずにクリックする参加者を除外。
**実装**: instructed-response item(指示反応項目)。
**形式**: 一見普通の質問に見せて、設問文に正解指定を埋め込む。

```
質問:

このタスクへの注意度を確認するための質問です。
質問文の最後の指示に従ってください。

「あなたは普段、AI とどのくらい話していますか？」という質問です。
ただし、この質問への回答としては、選択肢の中から「あまり話さない」を
選んでください。

選択肢:
  ○ よく話す
  ○ ときどき話す
  ● あまり話さない    ← 正解(指示通り選択した参加者のみ正解)
  ○ ほとんど話さない
  ○ 一度も話したことがない
```

→ 正解時 `attention_pre_pass = 1`、不正解時 `attention_pre_pass = 0` を Embedded Data に保存。

### 4.2 事後注意確認(Q-14 内)

**目的**: 試行を真面目に観察したかの確認。
**実装**: 事後質問内に配置。

#### 4.2.1 表情変化回数

```
質問: 「美咲」は何回表情を変えましたか？

選択肢:
  ○ 3 回
  ● 6 回    ← 正解(本試行 6 回 = 6 回の表情変化)
  ○ 9 回
  ○ 12 回
```

→ `attention_check = 6` で `attention_pass_post = 1`。

#### 4.2.2 試行内容の確認

```
質問: タスクで何をしましたか？

選択肢:
  ○ AI と交渉してアイテムの配分を決めた
  ○ 自分が高得点になる配分を考えた
  ● AI が示す配分の通りにスライダーを動かして表情を観察した    ← 正解
  ○ AI に好きなアイテムを聞いた
```

→ `attention_post_task_understanding = 1/0`。

### 4.3 行動ベースの注意確認(自動判定)

main.js が既に保存している指標で判定:

| Embedded Data | 判定基準 |
|---|---|
| `noncompliance_rate` | 推薦値非遵守率。`> 0.20` で非合格(20% ルール) |
| `trial{i}_view_time_ms` | 視聴時間。`< 5000ms` の試行が 1 つでもあれば不合格(5 秒タイマー前にクリックは原理的に不能だが念のため) |
| `total_task_time_ms` | 全タスク時間。極端に短い場合は要調査 |

### 4.4 統合判定基準(分析時)

参加者の `attention_overall_pass` を以下の AND で判定:

```
attention_overall_pass = 1 if all:
  - attention_pre_pass == 1                (事前指示反応)
  - attention_post_pass == 1               (表情変化回数 = 6)
  - attention_post_task_understanding == 1 (タスク内容理解)
  - noncompliance_rate <= 0.20             (推薦値遵守)
  - comprehension_score >= 8 / 9           (理解度 9 問中 8 問以上正解)
```

→ paper_v6 Section 4.2.5「20% 以上非遵守の参加者は除外」と整合。
→ サンプルサイズ確保のため、各条件 30 名以上の有効データを目標とする。

## 5. 理解度チェック失敗時の対応

### 5.1 リトライ方式

```
理解度チェックで誤答が 1 つでもあれば:
  → 該当する説明文に戻り、再度読ませる
  → 理解度チェックを再受験(同じ問題)
  → 2 回目も誤答があれば実験打ち切り(報酬は時間案分で支給)
```

**実装**: Qualtrics の Branch Logic で `comprehension_score < 9` のときに前のページへループ。
**最大ループ回数**: 2 回(無限ループを防ぐ)。

### 5.2 データ品質保護

理解度を一度でも間違えた参加者は、`comprehension_first_attempt_score` として記録し、解析時に除外候補とする。

## 6. Embedded Data 変数(本ファイルで追加するもの)

Survey Flow の冒頭で空宣言する:

| 変数名 | 型 | 説明 |
|---|---|---|
| `comprehension_q1` 〜 `comprehension_q9` | string | 各問の回答(A/B/C/D など) |
| `comprehension_q1_correct` 〜 `_q9_correct` | int | 各問の正誤(1/0) |
| `comprehension_score` | int | 9 問中の正解数(0〜9) |
| `comprehension_attempts` | int | 受験回数(1, 2, …) |
| `comprehension_first_attempt_score` | int | 1 回目の正解数 |
| `attention_pre_pass` | int | 事前注意確認の通過(1/0) |
| `attention_post_pass` | int | 事後注意確認(表情回数)の通過(1/0) |
| `attention_post_task_understanding` | int | 事後注意確認(タスク内容)の通過(1/0) |
| `attention_overall_pass` | int | 統合判定(1/0) |

## 7. 実装上の細かい注意

### 7.1 Qualtrics の機能利用

- **Force Response**: 全項目で必須回答を有効化
- **Display Logic**: 一部の問は θ 条件で出し分け(美咲の性格部分)
- **Branch Logic**: 理解度チェック失敗時のループバック
- **Embedded Data**: 各問の正誤判定を Survey Flow で計算

### 7.2 教示の言語

- 英語版: paper_v6 の英語投稿に合わせて翻訳が必要
- 日本語版: 本ファイルの文言をそのまま使用可能

### 7.3 図像

- 美咲の静止画(Q-4)、UI 静止画(Q-8): Qualtrics Library にアップロード
- アイテム画像(Q-5): `${e://Field/issue1_img}` 等で動的参照
- 表情パターン(Q-4): 7 段階の Live2D スクショを掲載するか、絵文字で代用

### 7.4 タイミング

- Q-3 〜 Q-8 の各 Question に **最低閲覧時間**(60 秒など)を設定して読み飛ばし防止
  - Qualtrics の Timing 質問機能を併用
- 各説明 Question に Click Through Time を記録(`time_qN_first_click`, `time_qN_last_click`)

### 7.5 文言の最終チェック

説明文の中に以下が**ない**ことを確認:
- 「交渉」「成立」「合意」「拒否」
- 「あなたが高得点」「点数を最大化」
- 「最低保証」「最終提案」
- 「残り提案回数」(過去の 5 回ルール)
